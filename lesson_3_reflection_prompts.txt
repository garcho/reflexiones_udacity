When would you want to use a remote repository rather than keeping all your work
local?

    Cuando se trabaja en el mismo proyecto desde diferentes lugares y computadores y cuando se trabaja con más personas 

Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?

    Porque es importante que los cambios guardados no sean al asar, es cada commit tenga un sentido que amerite guardar el cambio. 

Describe the differences between forks, clones, and branches.  When would you
use one instead of another?

    Fork es copiar el proyecto de un tercero para trabajarlo uno. Clonar es copiar un proyecto propio a un computador. Branches son ramans donde uno puede realizar trabajo sin afectar cada rama. 
    
What is the benefit of having a copy of the last known state of the remote
stored locally?

    Identificar con facilidad los cambios realizados por otras personas. 

How would you collaborate without using Git or GitHub?  What would be easier,
and what would be harder?

    Sencillo estar al día de los cambio y revisaron. Difícil mater un sentido de proposito 	común con los colaboradores. 

When would you want to make changes in a separate branch rather than directly in
master?  What benefits does each approach have?

    Cuando el código en master es el publico y más estable de todos y la rama (branch) es algo que se está propagando y no es definitivo. Se es definitivo cuando se une con master.
